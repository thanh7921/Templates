\section{Project Details}

\subsection{Simple RISC-V pipelined processor on Zynq SoC (FPGA)}
\cvlistitem{Basic RV32I intruction set, 6-stage pipeline with forwarding. Program binary is stored in BRAM.}
\cvlistitem{Tested and passed by RISCOF - The RISC-V Compatibility Framework.}
\cvlistitem{Simple printf() and scanf() functions, writing to custom memory-mapped UART IO. Design is assessed with Dhrystone benchmark. Text-based snake game is also ported to run on UART IO.}

\subsection{TCP/UDP packet parsing/encapsulating module (FPGA)}
\cvlistitem{Take in 32 bit word stream of IPv4 data frame, parse to extract various field of the
assumed contained UDP/TCP datagram, storing application data into a cache.}
\cvlistitem{Or given application data within the cache, perform encapsulation to create a IPv4
packet with chosen protocol (UDP or TCP) and field values.}
\cvlistitem{Verified functionality through simulation and visual inspection of output values for various network packets.}

\subsection{Simple 7-segment clock (FPGA)}
\cvlistitem{Clock displaying time of date in 24-hour format, up to the second. Can change time with button.}
\cvlistitem{Implemented on the DE-10 Lite development board, using Altera tool-chain.}
\cvlistitem{Verified by simulation. The final design is also tested in hardware and performed satisfactorily.}

\subsection{NFC + Facial Recognition Attendance System (Raspberry Pi}
\cvlistitem{Admittedly a fail project, trying to use NFC card (cheap MiFare), associate that
with face model data of each individual store on the Raspberry Pi to take attendance.}
\cvlistitem{OpenCV model chosen prioritize speed, since using Raspberry Pi.}
\cvlistitem{Technical reason for failing: insufficient memory on the Raspberry Pi, I should
have bought the version with more RAM.}

\subsection{Simplified smart house implementation with Yolo:bit (ESP32)}
\cvlistitem{Simplified programming of ESP32 chip (on Yolo:bit), running MicroPython.}
\cvlistitem{Setup AWS EC2 server to act as MQTT broker.}
\cvlistitem{Allow for remote control of devices, automatic activation based on environmental conditions.}

\subsection{Simple traffic light system with pedestrian yielding (STM32)}
\cvlistitem{Final project for the Embedded course back in university. Using STM32 dev-board and simple LEDs/buttons for the system implementation.}
\cvlistitem{Custom scheduler (support priority, no preemption) with binary heap priority queue. Timing using built-in timers. Button de-bouncing and all task scheduling uses the custom scheduler.}
